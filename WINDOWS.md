# 🪟 Запуск DSU на Windows

Подробная инструкция по установке и запуску Device Setup Utility на Windows.

---

## 📋 Требования

- **Windows 10/11** (64-bit)
- **Python 3.11+** (если запускаете из исходников)
- **Go 1.21+** (опционально, для сборки firmware модуля)

---

## ⚡ Способ 1: Готовый EXE (Самый простой)

### Шаг 1: Скачать
1. Перейдите в [Releases](https://github.com/igorynos/DSU/releases)
2. Скачайте `DSU.exe` из последнего релиза

### Шаг 2: Запустить
```cmd
DSU.exe
```

**Готово!** Программа запустится с графическим интерфейсом.

---

## 🐍 Способ 2: Из исходников (Рекомендуется для разработки)

### Шаг 1: Установить Python

1. Скачайте Python 3.11+ с [python.org](https://www.python.org/downloads/)
2. При установке **обязательно отметьте** "Add Python to PATH"
3. Проверьте установку:
```cmd
python --version
```

### Шаг 2: Распаковать архив

```cmd
cd D:\project
tar -xzf DSU_20260330_011846.tar.gz
cd DSU
```

Или используйте 7-Zip / WinRAR для распаковки `.tar.gz` файла.

### Шаг 3: Создать виртуальное окружение

```cmd
python -m venv venv
```

### Шаг 4: Активировать виртуальное окружение

```cmd
venv\Scripts\activate
```

После активации в командной строке появится `(venv)`:
```
(venv) D:\project\DSU>
```

### Шаг 5: Установить зависимости

```cmd
pip install -r requirements.txt
```

### Шаг 6: Запустить программу

**Современный интерфейс:**
```cmd
python app_modern.py
```

**Классический интерфейс:**
```cmd
python app.py
```

---

## 🔨 Способ 3: Собрать EXE самостоятельно

### Требования
- Python 3.11+
- Go 1.21+ (опционально)

### Шаг 1: Подготовить исходники

```cmd
cd D:\project\DSU
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
```

### Шаг 2: Собрать Go модуль (опционально, для ускорения прошивки)

```cmd
cd go-firmware
go build -o firmware.exe -ldflags="-s -w" firmware.go
cd ..
```

### Шаг 3: Собрать EXE

```cmd
build_exe.bat
```

Или вручную:
```cmd
pyinstaller DSU.spec
```

### Шаг 4: Найти EXE

```cmd
dist\DSU.exe
```

## 🎮 Использование

### Запуск

1. Дважды кликните `DSU.exe` или запустите через командную строку
2. Откроется графический интерфейс

### Добавление тестовых устройств

Для тестирования без реальных контроллеров:

1. Откройте меню **"Инструменты"**
2. Выберите **"Тестовые устройства"**
3. Нажмите **"Добавить 5 тестовых устройств"**

Тестовые устройства появятся в таблице с IP 192.168.0.100-104.

### Настройка устройства

1. **Выберите устройство** в таблице (кнопка "Выбрать")
2. **Отредактируйте параметры** в правой панели:
   - Имя устройства
   - IP адрес
   - Маска подсети
   - Шлюз
   - Порт
3. Нажмите **"💾 Применить настройки"**

### Прошивка устройства

1. Выберите устройство в таблице
2. Нажмите **"📦 Прошить устройство"**
3. Выберите файл прошивки (`.fw`)
4. Дождитесь завершения (прогресс-бар покажет статус)

## 🔧 Troubleshooting

### Ошибка: "python is not recognized"

**Решение**: Python не добавлен в PATH
1. Переустановите Python
2. Отметьте "Add Python to PATH" при установке
3. Или добавьте вручную: `C:\Python311\` и `C:\Python311\Scripts\`

### Ошибка: "No module named 'tkinter'"

**Решение**: Tkinter не установлен
1. Переустановите Python
2. При установке выберите "tcl/tk and IDLE"

### Ошибка: "ModuleNotFoundError: No module named 'keyboard'"

**Решение**: Зависимости не установлены
```cmd
venv\Scripts\activate
pip install -r requirements.txt
```

### GUI не отображается / черное окно

**Решение**: Проблема с графическим драйвером
1. Обновите драйверы видеокарты
2. Попробуйте запустить от имени администратора
3. Отключите антивирус временно

### Ошибка: "Permission denied" для broadcast

**Решение**: Запустите от имени администратора
1. Правый клик на `DSU.exe`
2. Выберите "Запуск от имени администратора"

### Антивирус блокирует EXE

**Решение**: Добавьте в исключения
1. PyInstaller EXE могут вызывать ложные срабатывания
2. Добавьте `DSU.exe` в исключения антивируса
3. Или соберите EXE самостоятельно из исходников

## 📡 Сетевые настройки

### Firewall

DSU использует UDP порты:
- **1770** - Locator (широковещательный поиск устройств)
- **1775** - ElUDP (управление устройствами)

Если Windows Firewall блокирует:
1. Откройте "Брандмауэр Windows"
2. Разрешите `DSU.exe` для частных и общедоступных сетей
3. Или добавьте правила для портов 1770 и 1775 UDP

### Broadcast в локальной сети

Для работы с реальными устройствами:
1. Подключитесь к той же локальной сети, что и контроллеры
2. Убедитесь, что IP адрес в той же подсети (например, 192.168.1.x)
3. Отключите VPN, если он мешает broadcast

## 🚀 Горячие клавиши

**Важно**: Горячие клавиши работают только при запуске от имени администратора

- `Ctrl+0` до `Ctrl+9` - Добавить/удалить тестовые устройства
- `Ctrl+R` - Обновить список устройств
- `Ctrl+Q` - Выход
